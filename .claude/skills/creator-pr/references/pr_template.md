# PR Description Template

This template is used by the PR Creator skill to generate PR descriptions automatically.

## Template Structure

```markdown
## Summary
[1-2 sentence overview of what this PR does]

## Changes
- [Bullet point for each major change]
- [Include files modified, functions added, behavior changed]

## Test plan
[How to verify these changes work correctly]
[Include manual testing steps or test cases]

---
Generated by PR Creator skill
```

## Template Fields

### Summary
- One sentence that explains what the PR accomplishes
- Keep it clear and non-technical when possible
- Example: "Adds responsive viewport switching to the component preview panel"

### Changes
- Bullet list of concrete changes
- Include file paths for clarity
- Be specific about what was added/modified/removed
- Example:
  ```
  - Modified src/components/LivePreview.tsx: Add viewport state and buttons
  - Updated src/App.css: Add styles for viewport controls and responsive preview
  ```

### Test plan
- How to manually verify the changes work
- List specific steps
- Include edge cases if relevant
- Example:
  ```
  1. Generate a component
  2. Click the three viewport buttons (📱 Mobile, 📱 Tablet, 🖥 Desktop)
  3. Verify preview width changes for each viewport
  4. Verify selected button is highlighted in orange
  5. Test on different screen sizes
  ```

---

## Auto-Population Rules

The PR Creator skill populates this template based on:

1. **Summary**: From PR title + commit message first line
2. **Changes**: From file names in `git diff --stat` + commit body
3. **Test plan**: From commit message (if it includes testing info) + inferred based on file types
   - React components → "Test rendering and interaction in browser"
   - CSS files → "Test styling on different viewports"
   - Backend changes → "Run test suite"

If the user provides a custom description, use that instead of auto-generating.
