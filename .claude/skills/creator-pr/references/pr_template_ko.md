# PR 설명 템플릿 (한국어)

## 템플릿 구조

```markdown
## 요약
[이 PR이 해결하는 내용을 1-2문장으로 설명]

## 변경사항
- [주요 변경 사항 1]
- [주요 변경 사항 2]
- [수정된 파일, 추가된 기능, 변경된 동작 등]

## 테스트 방법
[변경사항이 정상 작동하는지 확인하는 방법]
[수동 테스트 단계 또는 테스트 케이스 명시]

---
creator-pr 스킬로 생성됨
```

## 템플릿 필드 가이드

### 요약
- PR이 무엇을 하는지 명확하게 한 문장으로
- 기술적이지 않게 유지
- 예: "컴포넌트 미리보기에 반응형 뷰포트 전환 버튼 추가"

### 변경사항
- 구체적인 변경 항목을 bullet list로
- 수정된 파일 경로 포함
- 예시:
  ```
  - src/components/LivePreview.tsx 수정: viewport 상태 및 버튼 추가
  - src/App.css 업데이트: viewport 버튼 스타일 추가
  ```

### 테스트 방법
- 실제 테스트 단계를 단계별로 작성
- 엣지 케이스 포함
- 예시:
  ```
  1. 컴포넌트 생성
  2. 미리보기 헤더의 3개 뷰포트 버튼 클릭
  3. 각 버튼마다 미리보기 너비가 변경되는지 확인
  4. 선택된 버튼이 주황색으로 강조되는지 확인
  ```

## 자동 채우기 규칙

creator-pr 스킬에서 다음을 기반으로 자동 생성합니다:

1. **요약**: PR 제목 + 커밋 메시지 첫 줄
2. **변경사항**: `git diff --stat`의 파일명 + 커밋 본문
3. **테스트 방법**: 커밋 메시지(있는 경우) + 수정 파일 타입 기반 추론
   - React 컴포넌트 → "브라우저에서 렌더링 및 상호작용 테스트"
   - CSS 파일 → "다양한 화면 크기에서 스타일링 확인"
   - 백엔드 → "테스트 스위트 실행"

사용자가 커스텀 설명을 제공하면 그 대신 사용됩니다.
