# PR Creator

Automate the process of creating pull requests on GitHub. This skill takes your current git changes, generates a descriptive PR, creates a feature branch if needed, and opens it on GitHub using the `gh` CLI.

## What This Skill Does

1. **Analyzes changes**: Reads current `git diff` to understand what's changed
2. **Generates description**: Creates PR title and body based on commit messages and file changes
3. **Creates branch**: Generates a feature branch name (e.g., `feature/responsive-view`) and switches to it if needed
4. **Commits changes**: Stages and commits any uncommitted changes
5. **Opens PR**: Uses `gh pr create` to open the PR on GitHub
6. **Returns link**: Provides the PR URL for sharing

## When to Use

- User says "create a PR" or "make a pull request"
- User wants to submit changes to GitHub
- User has completed a feature and wants to open it for review
- Work-in-progress needs to be pushed as a draft PR

## Workflow

### Input: What the User Provides

The user should provide:
- **PR Title** (required): Short, descriptive title (e.g., "Add responsive view to component preview")
- **PR Description** (optional): Additional context or testing notes
- **Branch name** (optional): Custom branch name; auto-generated if not provided

**Example user prompts:**
```
"Create a PR for the responsive view feature"
"Open a PR: title='Fix API timeout handling', description='Increases timeout from 5s to 15s'"
"PR with branch feature/auth-flow"
```

### Steps the Skill Performs

#### Step 1: Verify Git Status

```bash
git status
git diff --stat
```

Check if there are uncommitted changes. If none, inform the user there's nothing to commit.

#### Step 2: Generate Branch Name

If the user doesn't provide one, derive it from the PR title:
- Input: "Add responsive view to component preview"
- Output: `feature/add-responsive-view`

Rules:
- Prefix with `feature/` (or `fix/`, `docs/` based on commit message type)
- Lowercase, replace spaces with hyphens
- Max 50 characters total

#### Step 3: Create and Switch Branch

```bash
git checkout -b <branch-name>
```

If branch already exists, checkout without creating.

#### Step 4: Stage and Commit Changes

```bash
git add .
git commit -m "<conventional commit message>"
```

Generate commit message:
- Analyze the PR title to infer the type (feat/fix/docs/style/refactor/chore)
- Format: `<type>: <message>`
- Example: `feat: add responsive view buttons to preview header`

#### Step 5: Generate PR Description

Use this template (`references/pr_template.md`):

```markdown
## Summary
[1-2 sentence summary of the change]

## Changes
- [Bullet list of key changes]

## Test plan
[How to verify the changes work]

Generated by PR Creator
```

Populate based on:
- Git commit messages
- File names changed (e.g., "Modified src/components/LivePreview.tsx")
- Diff content (identify what was added/removed)

#### Step 6: Push and Create PR

```bash
git push origin <branch-name>
gh pr create --title "<title>" --body "<body>" --draft
```

Use `--draft` flag to create a draft PR (user can mark ready).

Return the PR URL.

## Error Handling

- **No changes**: "No uncommitted changes found. Create some changes first, then run this skill again."
- **Detached HEAD**: "Cannot create PR from detached HEAD. Switch to a branch first."
- **No GitHub credentials**: "GitHub CLI is not authenticated. Run `gh auth login` and try again."
- **Branch conflicts**: "Branch already exists. Please resolve conflicts or use a different branch name."

## Configuration

This skill uses:
- **git**: Version control (built-in on most systems)
- **gh**: GitHub CLI (install via `brew install gh` or https://cli.github.com)

User must be authenticated: `gh auth login`
